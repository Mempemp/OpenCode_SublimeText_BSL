# OpenCode + Sublime

У тебя есть AI-агент (OpenCode), который пишет код. Но руками его править неудобно — нет нормального редактора. А Sublime — быстрый, лёгкий, с кучей фич. Этот пакет склеивает их в одну связку: агент генерит → ты допиливаешь в редакторе → агент работает дальше.

## Что внутри

- **Две слэш-команды** для OpenCode: открыть файлы в Sublime по имени или по гиту
- **Python-скрипт с умным поиском** — не надо писать полный путь, можно часть имени или только название файла
- **LSP для 1С/BSL**: ошибки подсвечиваются, автодополнение, прыжки по коду — всё как в EDT
- **Сервер BSL**: `bsl-language-server` от 1c-syntax, скачивается при установке

## Что нужно

- Windows 10/11
- OpenCode (уже стоит)
- Sublime Text 4 (уже стоит)
- Python 3 (уже есть в системе)
- Git
- Java 17+

## Что куда встанет

| После установки появится | Где |
|---|---|
| Команда `/OpenEditor` | В OpenCode — набираешь в чате |
| Команда `/OpenEditorDiffs` | В OpenCode — набираешь в чате |
| Подсветка и LSP для `.bsl` | В Sublime — само заработает |
| `bsl-language-server` | `~/tools/bsl-lsp/` (скачивается при установке) |

## Как пользоваться

### В OpenCode — `/OpenEditor`

Можно указать полный путь, а можно только часть — скрипт найдёт сам:

```
/OpenEditor CommonModules/АВТ_Интеграция/Ext/Module.bsl
/OpenEditor Module.bsl
/OpenEditor Интеграция
/OpenEditor src/main.bsl src/utils.bsl
```

**Как ищет:** точный путь → `git ls-files` по всем файлам проекта → дополняет `.bsl`/`.xml` если нет расширения → сканирует файловую систему. Открывает первый совпавший.

### В OpenCode — `/OpenEditorDiffs`

Открывает все изменённые файлы из git (staged + unstaged, кроме удалённых):

```
/OpenEditorDiffs
/OpenEditorDiffs *.bsl
```

**Важно:** пути — относительно корня проекта (там где лежит `.git`).

### В Sublime

Открыл `.bsl` файл — и всё работает из коробки:

| Что делаешь | Как |
|---|---|
| Посмотреть ошибки | Сами красным подчёркнуты |
| Дополнить слово | `Ctrl+Space` |
| Прыгнуть в процедуру | `F12` или `Ctrl+Click` |
| Где используется | `Shift+F12` |
| Список всех процедур | `Ctrl+R` |
| Переименовать | `F2` |
| Отформатировать | `Ctrl+Shift+P` → `LSP: Format` |

Плюс специфичное для 1С: когнитивная и цикломатическая сложность подсвечиваются.

### Как выглядит рабочий день

1. Сидишь в OpenCode, общаешься с агентом, он что-то нагенерил
2. Набираешь `/OpenEditorDiffs *.bsl` — Sublime открывает всё что поменялось
3. Смотришь диффы, правишь руками, пользуешься `F12`, `Ctrl+Space`
4. Сохраняешь (`Ctrl+S`), возвращаешься в OpenCode
5. «Я поправил, продолжай» — агент едет дальше

## Установка

```powershell
powershell -ExecutionPolicy Bypass -File install.ps1
```

Потом перезапусти OpenCode и Sublime. Всё.

## Настройка

После установки конфиг лежит в `~/.config/opencode/scripts/config.json`:

```json
{
  "sublime_exe": "C:/Program Files/Sublime Text/subl.exe",
  "java_exe":     "java",
  "lsp_jar_dir":  "tools/bsl-lsp",
  "lsp_jar_glob": "bsl-language-server-*-exec.jar"
}
```

| Параметр | Что делает |
|----------|-----------|
| `sublime_exe` | Полный путь к `subl.exe` (если Sublime не в стандартной папке) |
| `java_exe` | Путь к Java (если `java` не в PATH) |
| `lsp_jar_dir` | Папка с jar-файлом BSL-сервера (относительно `$HOME`) |
| `lsp_jar_glob` | Маска имени jar-файла (менять только при обновлении сервера) |
